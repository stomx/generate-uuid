# Korean Commit Skill

이 스킬은 작업 내용을 유의미한 단위로 나누고, 커밋 컨벤션에 따라 한국어로 커밋 메시지를 작성하는 지침을 제공합니다.

## 지침

1. **작업 단위 분할**: 변경 사항을 논리적으로 독립된 기능이나 수정 단위로 나눕니다.
2. **커밋 컨벤션 준수**: `feat:`, `fix:`, `style:`, `refactor:`, `chore:`, `docs:` 등의 접두사를 사용합니다.
3. **한국어 메시지**: 커밋 메시지의 제목과 본문은 모두 한국어로 작성합니다.
4. **구체적인 설명**: 메시지는 무엇을 왜 변경했는지 알 수 있도록 작성합니다.
